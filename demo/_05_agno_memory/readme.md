# 记忆系统

### 作用
* 上下文连续性
* 个性化服务
* 任务追踪
* 学习与改进

### 实现架构
* MemoryManager，Agno通过这个类，来实现记忆的功能

### 核心的关键
**是基于user_id的记忆：**
* 这里的user是一个抽象的概念，可以认为是记忆的分区，用来做隔离用的；而不是只能存用户相关的信息
* 所有的记忆的存取，都应该基于user_id进行隔离，如果不传入，则默认使用"default"作为user_id`

### 记忆模式有两种：
* **自动记忆**：每次run()即每轮对话后台额外调用一次LLM提炼并存储，Agent无感，不知道后台有进行存储动作，所以也无法主动删除记忆
* **Agentic模式**：给Agent提供Mem相关tool，Agent知道自己拥有了tool，所以在对话中会自主调用tool进行记忆，也可以调用删除tool删除记忆


### 记忆系统检索
* 注意：记忆系统检索不是向量检索！记忆系统本质上就是把db中的信息读取来发给大模型，所以它适用于一些简单但是重要的信息。

### 记忆共享的抓手：user_id
* 其实还是上面提到的隔离，通过这个抓手，可以实现：多轮对话共享记忆、多Agent之间共享记忆

### 新版的记忆
* 记忆的设计都是按user_id来隔离的，设计上比较粗糙
* 新版的记忆系统，有一个叫做LearningMachine体系，需要更高级的功能，可以使用这个体系
